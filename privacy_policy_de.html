3. Wie wir Ihre Daten verwenden

3.1 Echtzeitübersetzung

Wenn Sie die Übersetzungsfunktion verwenden:

  1. Der von Ihnen getippte Text wird an unseren Cloudflare Worker-Server gesendet (sicheres Relais)
  2. Der Worker leitet den Text zur Übersetzung an OpenAI weiter
  3. Die Übersetzung wird an die App zurückgesendet
  4. ⚠️ WICHTIG: Der Text wird NIEMALS auf unseren Servern oder bei OpenAI gespeichert (nur Durchleitung im Speicher)

3.2 Token-Backup und Wiederherstellung

Gekaufte Token werden auf Firebase (Google Cloud) gespeichert, um Ihnen Folgendes zu ermöglichen:

  • ✅ Wiederherstellung von Token bei Neuinstallation der App
  • ✅ Synchronisierung von Token über mehrere Geräte
  • ✅ Beibehaltung des Guthabens auch nach Gerätewechsel
⚠️ GOOGLE-KONTO ERFORDERLICH: Die Wiederherstellung von Token auf mehreren Geräten funktioniert NUR, wenn alle Geräte dasselbe Google Play-Konto verwenden, mit dem Sie die Käufe getätigt haben. Diese Anforderung wird von der Google Play Billing API aus Sicherheitsgründen und zur Betrugsprävention durchgesetzt. Wenn Sie Ihr Google-Konto wechseln, müssen Sie die Token im neuen Konto erneut kaufen.

3.3 Anti-Betrugs-System für Token

⚠️ WICHTIG - WIE DIE TOKEN-WIEDERHERSTELLUNG FUNKTIONIERT:

Um Missbrauch und Betrug zu verhindern, implementiert kAIboard ein Anti-Betrugs-System, bei dem nur EIN Gerät gleichzeitig die gekauften Token verwenden kann.

Verhalten während der Wiederherstellung:

  1. Gerät A kauft 50.000 Token → Gerät A ist „aktiv" ✅
  2. Gerät B (gleiches Google Play-Konto) klickt „Käufe wiederherstellen" → Gerät B wird „aktiv" ✅
  3. Gerät A wird automatisch „invalidiert" → Token auf Gerät A werden auf Null gesetzt ❌
  4. Um Gerät A zu reaktivieren: Klicken Sie erneut auf „Käufe wiederherstellen" auf Gerät A

Für das Anti-Betrugs-System gespeicherte Metadaten:

  • active_device: ID des aktuell aktiven Geräts
  • invalidated_at: Zeitstempel, wann ein Gerät deaktiviert wurde
  • invalidated_reason: Grund für die Deaktivierung (z.B. „migrated_to_android_xxx")
  • migrated_from: Ursprungsgerät der Token
  • migration_timestamp: Datum/Uhrzeit der Migration

Rechtsgrundlage: Berechtigtes Interesse zur Verhinderung von Betrug und Missbrauch des Kaufsystems (Art. 6 Abs. 1 lit. f DSGVO)

💡 HINWEIS: Dieses System stellt sicher, dass gekaufte Token nicht betrügerisch auf mehrere Geräte dupliziert werden. Die Token-Migration ist irreversibel: Wenn Sie auf Gerät B wiederherstellen, verliert Gerät A die Token dauerhaft, bis Sie eine neue Wiederherstellung auf Gerät A durchführen.

3.4 Kaufüberprüfung

Die Kaufhistorie wird gespeichert, um:

  • ✅ Gültigkeit von In-App-Käufen zu überprüfen
  • ✅ Betrug und Missbrauch zu verhindern
  • ✅ Rückerstattungen und Streitigkeiten mit Google Play zu verwalten

Wenn Sie ein aktives Monatsabonnement haben, wird die Löschung der Cloud-Daten vorübergehend blockiert, bis Sie das Abonnement kündigen.

  1. Kündigen Sie das Abonnement über den Google Play Store
  2. Warten Sie bis zum Ende des aktuellen Abrechnungszeitraums
  3. Löschen Sie die Cloud-Daten über die App

Sofortige Löschung: Die App leitet Sie automatisch zu Google Play weiter, um das Abonnement zu verwalten. Nach der Kündigung ist die Datenlöschung sofort verfügbar.

Rechtsgrundlage der Blockierung: Art. 17.3.b DSGVO - „Das Recht auf Löschung gilt nicht, soweit die Verarbeitung erforderlich ist zur Erfüllung eines Vertrags, dessen Vertragspartei die betroffene Person ist".
